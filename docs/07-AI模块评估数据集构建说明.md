## 1. 概述

### 1.1 目的

为系统 6 个核心 AI 模块的离线评估构建标准化测试数据集，确保评估结果可复现、可量化、可追溯。

### 1.2 数据集总览

| 数据集 | 代号 | 规模 | 用途 |
|--------|------|------|------|
| NER测试集 | NER-500 | ≥500 句 | NLU 实体识别 F1 评估 |
| 意图分类测试集 | INT-500 | ≥500 句 | NLU 意图分类 Accuracy 评估 |
| 同义词测试集 | SYN-200 | 200 条 | 同义词归一化覆盖率 |
| Cypher 模板测试集 | CYP-100 | 100 问 | Text2Cypher 模板覆盖率 |
| 答案准确性测试集 | QA-100 | 100 问 | RAG 综合准确率 |
| DDI 测试集 | DDI-50 | 50 组药物对 | 药物相互作用检出率 |
| 安全性测试集 | SAF-30 | 30 条有害建议 | 答案校验安全拦截率 |
| 未知问题测试集 | UNK-30 | 30 条 | 置信度兜底触发率 |

## 3. 数据预处理流程

### 3.1 文本清洗

```
① 去除 HTML/XML 标签、特殊控制字符
② 统一标点符号（全角 → 半角）：，→ , 。→ . 等
③ 繁体中文 → 简体中文（OpenCC 或 zhconv）
④ 去除多余空白、连续换行
⑤ 统一日期/数字格式
⑥ 去除重复段落
```

### 3.2 分句与分词

```python
# 分句：基于标点符号和句法规则
import re
sentences = re.split(r'[。！？!?；;\n]+', text)

# 分词：jieba + 自定义医学词典
import jieba
jieba.load_userdict("medical_dict.txt")  # 加载医学专有名词
tokens = jieba.lcut(sentence)
```

**自定义医学词典内容**（示例）：
```
复方降压片 10 n
2型糖尿病 10 n
紧张性头痛 10 n
神经内科 10 n
头颅MRI 10 n
```

### 3.3 自动预标注

使用预训练的 BioBERT/RoBERTa 模型进行自动实体识别和关系预标注：
- 模型：`dmis-lab/biobert-v1.1`（HuggingFace）
- 实体类型：Disease, Drug, Symptom, Examination, Department, Population
- 输出格式：BIO 标注序列
- 生成初步标注结果供人工校对

## 4. 标注规范（BIO 体系）

### 4.1 标注格式

```
B-XXX  : 实体起始字符
I-XXX  : 实体内部字符
O      : 非实体字符
```

其中 XXX 为实体类型标识：
| 标识 | 实体类型 | 示例 |
|------|---------|------|
| DIS | 疾病 Disease | 高血压、偏头痛 |
| DRG | 药品 Drug | 复方降压片、阿司匹林 |
| SYM | 症状 Symptom | 头痛、发烧、恶心 |
| EXM | 检查 Examination | 头颅MRI、血糖检测 |
| DEP | 科室 Department | 神经内科、心血管内科 |
| POP | 人群 Population | 孕妇、老年人 |

### 4.2 标注示例

```
原句：高血压患者服用复方降压片后出现头痛恶心

标注：高    血    压    患    者    服    用    复    方    降    压    片    后    出    现    头    痛    恶    心
      B-DIS I-DIS I-DIS O     O     O     O     B-DRG I-DRG I-DRG I-DRG O     O     O     B-SYM I-SYM B-SYM I-SYM
```

### 4.3 实体 JSON 格式

```json
{
  "entity_id": "ent_hbp_001",
  "name": "高血压",
  "type": "Disease",
  "start_offset": 0,
  "end_offset": 3,
  "source_doc": "内科学第9版_第5章"
}
```

### 4.4 关系 JSON 格式

```json
{
  "relation_id": "rel_001",
  "head_entity_id": "ent_rsfj_001",
  "head_entity_name": "复方降压片",
  "relation_type": "treats",
  "tail_entity_id": "ent_hbp_001",
  "tail_entity_name": "高血压",
  "confidence": 0.96,
  "source": "国家基层高血压防治指南(2020)"
}
```

## 6. 各模块评估数据集设计

### 6.1 NER-500：实体识别测试集

**规模**：≥500 句医学问句
**覆盖**：6 类实体均衡分布，每种 ≥80 个标注样本
**问句构造方法**：

| 类别 | 示例问句 | 标注实体 |
|------|---------|---------|
| 疾病查询 | "高血压有什么症状" | 高血压(DIS) |
| 药品查询 | "阿司匹林的副作用是什么" | 阿司匹林(DRG) |
| 症状咨询 | "头痛发烧三天了怎么办" | 头痛(SYM), 发烧(SYM) |
| 检查建议 | "血糖检测需要空腹吗" | 血糖检测(EXM) |
| 科室推荐 | "应该挂什么科看偏头痛" | 偏头痛(DIS) |
| 人群限制 | "孕妇可以吃感冒药吗" | 孕妇(POP), 感冒药(DRG) |
| 混合多实体 | "高血压合并糖尿病吃什么药合适" | 高血压(DIS), 糖尿病(DIS) |

**输出文件**：`ner_test.jsonl`
```json
{"text": "高血压患者头痛怎么办", "labels": ["B-DIS","I-DIS","I-DIS","O","B-SYM","I-SYM","O","O","O"]}
```

### 6.2 INT-500：意图分类测试集

**规模**：≥500 句，5 类意图各 ≥100 句
**类别定义**：

| 意图 | 标识 | 示例问句 |
|------|------|---------|
| 症状咨询 | consult_symptom | "头晕恶心想吐是什么原因" |
| 用药咨询 | consult_drug | "布洛芬一天最多吃几次" |
| 疾病查询 | disease_query | "糖尿病的早期症状有哪些" |
| 禁忌查询 | contraindication_query | "高血压患者不能吃什么药" |
| 科室推荐 | dept_recommend | "胸闷气短应该挂什么科" |

**输出文件**：`intent_test.jsonl`
```json
{"text": "头晕恶心想吐是什么原因", "label": "consult_symptom"}
```

### 6.3 SYN-200：同义词测试集

**规模**：从 ≥3000 条同义词库中随机抽样 200 条
**覆盖**：覆盖全部 6 类实体
**测试方法**：输入口语词 → 检查返回的标准术语是否正确

| 口语词 | 标准术语 | 实体类型 |
|--------|---------|---------|
| 拉肚子 | 腹泻 | Symptom |
| 感冒药 | 复方氨酚烷胺 | Drug |
| 高血压 | 高血压病 | Disease |
| 拍片 | X线摄影 | Examination |
| 内科 | 内科学 | Department |

**输出文件**：`synonym_test.jsonl`
```json
{"colloquial": "拉肚子", "standard": "腹泻", "entity_type": "Symptom"}
```

### 6.4 CYP-100：Cypher 模板覆盖测试集

**规模**：100 问（5 类意图 × 20 问）
**构造方法**：
1. 每种意图构造 20 个变体问句（不同实体、不同表达方式）
2. 每道题标注：预期使用的模板编号、预期填充的实体参数
3. 统计模板命中率

| 意图 | 预期模板 | 测试问句 | 预期结果 |
|------|---------|---------|---------|
| consult_symptom | template_1 | "高血压有哪些症状" | MATCH (:Disease {name:"高血压"})-[:HAS_SYMPTOM]->(s) RETURN s |
| consult_drug | template_2 | "吃什么药治高血压" | MATCH (:Disease {name:"高血压"})-[:TREATED_BY]->(d) RETURN d |

**输出文件**：`cypher_test.jsonl`
```json
{
  "query": "高血压有哪些症状",
  "intent": "consult_symptom",
  "expected_template": "template_1",
  "expected_params": {"disease": "高血压"},
  "expected_cypher": "MATCH (d:Disease {name: '高血压'})-[:HAS_SYMPTOM]->(s:Symptom) RETURN s.name, s.description"
}
```

### 6.5 QA-100：答案准确性测试集

**规模**：100 问（覆盖全部 5 类意图）
**每题包含**：
- 问句
- 标准答案（由医学专家编写或从权威指南摘录）
- 预期知识来源
- 评分维度（准确性、完整性、安全性）

| # | 问句 | 标准答案要点 | 知识来源 |
|---|------|-------------|---------|

## 7. 质量控制机制

### 7.1 标注一致性检验

- **指标**：Cohen's Kappa 系数
- **方法**：标注员 A 和标注员 B 独立标注同批次 50 条数据
- **标准**：Kappa ≥ 0.85，低于此值的批次需返工重标
- **频率**：每个批次（每200条）进行一次

### 7.2 实体与关系校验

- 检查实体必填字段完整性（name, type, offset 不可为空）
- 验证关系头尾实体类型与预定义约束的匹配性
- 示例约束：`HAS_SYMPTOM` 关系只能从 Disease → Symptom

### 7.3 专家审核

- **审核人员**：副高及以上职称医学专家
- **抽样比例**：每批次随机抽取 10%
- **通过标准**：标注错误率 ≤ 5%
- **超标处理**：错误率 > 5% 的批次全面返工

### 7.4 数据集质量报告模板

```
数据集批次： NER-500-B001
标注日期： 2026-06-27
标注员A： 李XX（临床医学本科）
标注员B： 王XX（药学本科）
Cohen's Kappa： 0.91 ✅
专家审核抽样： 50条/500条 (10%)
错误率： 2.4% (12/500) ✅
处理结果： 通过，入库
```
