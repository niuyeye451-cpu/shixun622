| 数据库 | 定位 | 数据模型 | 规模 |
|--------|------|---------|------|
| MySQL | 业务数据持久化 | 关系型（表/行/列） | 16 张表 |
| Neo4j | 医学知识图谱 | 图（节点/关系/属性） | ≥10万节点 + ≥50万边 |
| Milvus | 语义向量检索 | 集合（向量+元数据） | 10万级向量 |
| Redis | 缓存 + 会话 + 限流 | 键值对（TTL过期） | 8GB 内存上限 |

## 二、MySQL — 业务数据库

### 2.1 ER总览（概要设计文档 第五部分）

系统共 16 个实体，分为 5 组：

```
用户体系: Patient, Doctor, Administrator, SmsCode, LoginLog
会话体系: Conversation, Message, Consultation, Registration
知识体系: Medical_Entity, Medical_Relation, Synonym
反馈体系: Feedback, UnknownQuestion, Knowledge_Task, KnowledgeVersion
审计运营: OperationLog, Statistics
```

### 2.2 表结构详解

#### 用户体系（5 张表）

**Patient（患者）**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| patient_id | VARCHAR(32) | PK | UUID 前缀 `pat_` |
| user_name | VARCHAR(50) | NOT NULL | 用户名，默认手机尾号4位 |
| phone | VARCHAR(20) | UNIQUE, INDEX, NOT NULL | 手机号，登录凭证 |
| password_hash | VARCHAR(100) | NULLABLE | bcrypt 哈希，短信验证码登录时为 NULL |
| avatar | VARCHAR(255) | NULLABLE | 头像 URL |
| gender | VARCHAR(10) | NULLABLE | 性别 |
| age | INTEGER | NULLABLE | 年龄 |
| address | VARCHAR(255) | NULLABLE | 地址 |
| blood_type | VARCHAR(10) | NULLABLE | 血型 |
| allergy_history | TEXT | NULLABLE | 过敏史 JSON 数组 |
| medical_history | TEXT | NULLABLE | 既往病史 JSON 数组 |
| status | INTEGER | DEFAULT 1 | 1=正常, 0=禁用 |
| created_at | DATETIME | DEFAULT NOW() | 注册时间 |

关联关系：
```
Patient → [Conversation]  ←→  conversations  (一对多)
Patient → [Message]       ←→  messages       (一对多)
Patient → [Consultation]  ←→  consultations  (一对多)
Patient → [Registration]  ←→  registrations  (一对多)
Patient → [Feedback]      ←→  feedbacks      (一对多)
```

**Doctor（医师）**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| doctor_id | VARCHAR(32) | PK | UUID 前缀 `doc_` |
| user_name | VARCHAR(50) | NOT NULL | 用户名，管理员创建 |
| phone | VARCHAR(20) | UNIQUE, INDEX, NOT NULL | 手机号 |
| password_hash | VARCHAR(100) | NOT NULL | bcrypt 哈希 |
| department | VARCHAR(50) | NULLABLE | 所属科室 |
| title | VARCHAR(50) | NULLABLE | 职称（主任医师/副主任医师等） |
| hospital | VARCHAR(100) | NULLABLE | 所属医院 |
| specialty | VARCHAR(255) | NULLABLE | 专长领域 |
| introduction | TEXT | NULLABLE | 个人简介 |
| avatar | VARCHAR(255) | NULLABLE | 头像 |
| is_first_login | BOOLEAN | DEFAULT TRUE | 首次登录强制改密码 |
| status | INTEGER | DEFAULT 1 | 1=正常, 0=禁用 |
| created_at | DATETIME | DEFAULT NOW() | 创建时间 |

**Administrator（管理员）**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|

**SmsCode（短信验证码）**

| 字段 | 类型 | 约束 | 说明 |

## 三、Neo4j — 知识图谱

### 3.1 节点类型与属性

**6 类节点**

```
(:Disease)       — 蓝色 #165DFF
(:Drug)          — 绿色 #00B42A
(:Symptom)       — 红色 #F53F3F
(:Department)    — 橙色 #FF7D00
(:Examination)   — 紫色 #722ED1
(:Population)    — 青色 #14C9C9
```

**节点属性模板**

```cypher
-- Disease
CREATE (:Disease {
  name: '高血压',           -- 标准名称（唯一约束索引）
  icd10: 'I10',             -- ICD-10 编码
  description: '以体循环动脉血压升高为主要特征的临床综合征',
  aliases: ['高血压病', '原发性高血压'],
  source_version: 'v20260627-001',
  prevalence: '中国成人患病率 27.9%',
  risk_factors: ['高盐饮食', '肥胖', '遗传', '精神紧张']
});

-- Drug
CREATE (:Drug {
  name: '复方降压片',
  category: '降压药',
  atc_code: 'C02LB01',      -- ATC 药物编码
```

### 3.2 关系类型

**11 类核心关系**

| 关系类型 | 方向 | 语义 | 属性 | 线型（前端） |
|---------|------|------|------|------------|
| HAS_SYMPTOM | Disease→Symptom | 疾病表现症状 | probability(概率) | 实线 |
| TREATS | Drug→Disease | 药品治疗疾病 | efficacy, evidence_level | 实线 |
| TREATED_BY | Disease→Department | 疾病归属科室 | — | 实线 |
| BELONGS_TO | Symptom→Disease 或 Disease→Department | 从属关系 | — | 实线 |
| CONTRAINDICATED_FOR | Drug→Disease | 药品对某疾病禁忌 | reason | 红色虚线 |
| CONTRAINDICATED_WITH | Drug→Drug | 药物禁忌联用 | effect, mechanism, evidence | 红色虚线 |
| CAUTION_WITH | Drug→Drug | 谨慎联用 | effect, mechanism | 橙色虚线 |
| INTERACTS_WITH | Drug→Drug | 存在相互作用 | effect, mechanism | 黄色虚线 |
| CAUSES | Drug→Symptom | 引起不良反应 | probability, severity | 虚线 |
| NEED_CHECK | Disease→Examination | 确诊需检查 | is_routine(常规/特殊) | 虚线 |
| COMPLICATION | Disease→Disease | 并发症 | probability | 虚线 |
| APPLICABLE_TO | Drug→Population | 适用人群 | age_range, notes | 点线 |
| NOT_FOR | Drug→Population | 禁用人群 | reason | 红色虚线 |
| DO_EAT / NOT_EAT | Disease→Food | 饮食建议 | reason | 点线 |

**关系创建示例**

```cypher
-- 疾病 → 症状
MATCH (d:Disease {name: '高血压'}), (s:Symptom {name: '头痛'})
CREATE (d)-[:HAS_SYMPTOM {probability: 0.65}]->(s);

-- 药品 → 疾病
MATCH (drug:Drug {name: '复方降压片'}), (d:Disease {name: '高血压'})
CREATE (drug)-[:TREATS {efficacy: '显著', evidence_level: 'A级'}]->(d);

-- 药物禁忌联用
MATCH (a:Drug {name: '华法林'}), (b:Drug {name: '阿司匹林'})
CREATE (a)-[:CONTRAINDICATED_WITH {
  effect: '严重出血风险', 
  mechanism: '协同抗凝作用',
  evidence: 'DrugBank DB00398'
}]->(b);

-- 疾病→科室
MATCH (d:Disease {name: '高血压'}), (dept:Department {name: '心血管内科'})
CREATE (d)-[:TREATED_BY]->(dept);
```

### 3.3 常用查询模式

```cypher
-- 模式1: 症状查疾病 + 科室（导诊核心查询）
MATCH (s:Symptom {name: '头痛'})<-[:HAS_SYMPTOM]-(d:Disease)-[:TREATED_BY]->(dept:Department)
RETURN d.name AS disease, dept.name AS department;

-- 模式2: 疾病查药品
MATCH (d:Disease {name: '高血压'})<-[:TREATS]-(drug:Drug)
RETURN drug.name, drug.usage, drug.side_effects;

-- 模式3: 多药相互作用检测
MATCH (d1:Drug)-[r:CONTRAINDICATED_WITH|CAUTION_WITH|INTERACTS_WITH]-(d2:Drug)
WHERE d1.name IN ['华法林', '阿司匹林'] AND d2.name IN ['华法林', '阿司匹林']
  AND id(d1) < id(d2)
RETURN d1.name, d2.name, type(r) AS rel_type, r.effect;

-- 模式4: 多跳推理（疾病→并发症→检查→科室）
MATCH path = (d:Disease {name: '2型糖尿病'})-[:COMPLICATION|NEED_CHECK|TREATED_BY*1..3]-(related)
RETURN [n IN nodes(path) | n.name] AS path_nodes,
       [r IN relationships(path) | type(r)] AS path_edges
LIMIT 5;
```

## 五、Redis — 缓存数据库

### 5.1 键空间设计

| 命名空间 | Key 模式 | 数据类型 | TTL | 说明 |
|---------|---------|---------|-----|------|
| `qa:cache:` | `qa:cache:{md5(question)}` | JSON String | 3600s (1h) | 热门问题答案缓存 |
| `session:` | `session:{session_id}` | JSON String (List) | 1800s (30min) | 多轮对话上下文 |
| `token:` | `token:{user_id}` | String | JWT 过期时间 | JWT 令牌白名单 |
| `blacklist:` | `blacklist:{token}` | String ("1") | 令牌剩余有效期 | 主动登出失效令牌 |
| `ratelimit:` | `ratelimit:{user_id}:{minute}` | Integer (INCR) | 60s | API 限流计数器 |
| `synonym:` | `synonym:all` | Hash | 永久 (定时刷新) | 同义词热缓存 |
| `hot:` | `hot:entities:{date}` | Sorted Set | 86400s (24h) | 热点实体排行 |
| `lock:` | `lock:knowledge_update` | String (SETNX) | 60s | 知识更新分布式锁 |

### 5.2 会话管理

```python
# 多轮对话上下文（最近5轮）
key = f"session:{session_id}"
context = [
    {"role": "user", "content": "头痛怎么办"},
    {"role": "assistant", "content": "建议..."},
    {"role": "user", "content": "吃什么药"},
    {"role": "assistant", "content": "根据..."},
]
redis.setex(key, 1800, json.dumps(context[-10:]))  # 只保留最后10条消息
```

### 5.3 接口限流

```python
# 滑动窗口：每分钟最多 10 次
key = f"ratelimit:{user_id}:{datetime.now().strftime('%Y%m%d%H%M')}"
count = redis.incr(key)
redis.expire(key, 60)
if count > 10:
    raise HTTPException(429, "请求过于频繁，请稍后再试")
```
