## 二、核心数据文件夹（最重要）

### 1. medical_clean.json（51MB）

**文件内容**：
- 8807 条疾病记录的完整清洗数据
- 每条记录包含：疾病名、症状、药品、科室、检查、治疗方式、饮食建议、预防措施等

**对项目的作用**：
- 是整个系统的**数据底座**
- 图谱组用它提取节点属性（疾病的简介、病因、预防等）
- 后端用它做疾病详情查询

**给谁用**：
- 图谱组：作为 Disease 节点的属性来源
- 后端：疾病详情页面的数据来源

**使用方式**：
```python
# 读取示例
import json
with open("medical_clean.json", "r") as f:
    for line in f:
        disease = json.loads(line)
        # disease["name"] = "感冒"
        # disease["symptom"] = ["头痛", "发热", ...]
```

---

### 2. triplets.csv（29MB）

**文件内容**：
- 459,387 条三元组（头实体, 关系, 尾实体）
- 名称版（实体用名称而非ID）

**对项目的作用**：
- 是知识图谱的**边数据**
- 定义了疾病与症状、药品、科室等的关系
- 图谱组用它创建 Neo4j 的关系边

**给谁用**：
- 图谱组：`LOAD CSV` 批量导入 Neo4j
- 数据组：关系方向一致性校验

**文件格式**：
```
head,relation,relation_cn,tail
感冒,has_symptom,症状,头痛
感冒,need_check,诊断检查,血常规
感冒,common_drug,常用药品,感冒灵颗粒
```

---

### 3. triplets_with_id.csv（35MB）

**文件内容**：
- 459,387 条三元组（同上）
- **ID版**（实体用唯一ID而非名称）

**对项目的作用**：
- 是图谱组的**直接可用数据**
- 用ID做节点主键，比用名称更稳定、更高效
- 避免同名实体冲突问题

**给谁用**：
- 图谱组：直接 `LOAD CSV` 入库，无需再建名称→ID映射

**与 triplets.csv 的区别**：
```
# triplets.csv（名称版）
感冒,has_symptom,症状,头痛

# triplets_with_id.csv（ID版）
D00123,感冒,has_symptom,症状,S00456,头痛
```

## 四、同义词表文件夹

### 1. synonym_full.json（112KB）

**内容**：
- 疾病别名：1275 组（如 感冒→伤风）
- 症状别名：17 组（如 头疼→头痛）
- 药品别名：13 组（如 布洛芬→芬必得）

**作用**：
- 完整的同义词词典
- 可挂到图谱节点作为 `alias` 属性
- NLP组可用于 NER 词典扩充

**格式**：
```json
{
  "disease": {
    "感冒": ["伤风", "上呼吸道感染"],
    "矽肺": ["硅肺"]
  },
  "symptom": {
    "头痛": ["头疼"]
  }
}
```

---

### 2. synonym_alias.json（102KB）⭐ 核心

**内容**：2232 条 {别名: 标准名} 映射

**作用**：
- **问答召回的核心文件**
- 用户输入"头疼" → 映射为"头痛" → 再查图谱
- 用户输入"伤风" → 映射为"感冒" → 再查图谱

**给谁用**：
- 问答组：加载此文件，在查询前做同义词归一

**格式**：
```json
{
  "头疼": "头痛",
  "伤风": "感冒",
  "芬必得": "布洛芬",
  "潜水夫病": "减压病"
}
```

**使用示例**：
```python
import json
alias_map = json.load(open("synonym_alias.json"))
user_input = "头疼"
std_word = alias_map.get(user_input, user_input)  # → "头痛"
# 用 std_word 查图谱
```

## 五、ID映射文件夹

### 1. entity_id_map.json（4.4MB）⭐ 重要

**内容**：44650 个实体 → ID 的映射，含别名和频次

**作用**：
- **图谱节点的稳定主键**
- Neo4j 不应用 name 做主键，用 ID 更健壮
- 含频次信息，可用于推荐排序

**给谁用**：
- 图谱组：用 ID 建节点，name 作为属性
- 后端：查询结果可用 ID 精确引用实体

**格式**：
```json
{
  "Disease": {
    "感冒": {
      "id": "D00123",
      "alias": ["伤风", "上呼吸道感染"],
      "freq": 156
    }
  },
  "Symptom": {
    "头痛": {
      "id": "S00456",
      "alias": ["头疼"],
      "freq": 892
    }
  }
}
```

## 七、各文件对项目问答功能的支撑

| 用户问题类型 | 支撑文件 | 作用 |
|------------|---------|------|
| "感冒有什么症状？" | triplets.csv + Symptom.txt | 查 (感冒)-[:has_symptom]->(症状) |

| "感冒做什么检查？" | triplets.csv + Check.txt | 查 (感冒)-[:need_check]->(检查) |
| "感冒吃什么食物？" | triplets.csv + 宜吃食物.txt | 查 (感冒)-[:do_eat]->(食物) |
